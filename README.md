## Задача 1

### Текст задачи

На основе последовательности вещественных чисел получить последовательность целых чисел, отбросив дробную часть.



### Алгоритм решения

Программа начинает свое выполнение с функции main. Запрашивается у пользователя количество значений последовательности, которое сохраняется в значение count. / Выполняется проверка: если count меньше или равно 0, выводится сообщение «Ошибка: количество должно быть положительным!» и программа аварийно завершается с ошибкой через exit 1. Если count положительное число, Затем вызывается функция readFloatsSeq где формируется последовательность, которая дальше передается в функцию seq.map для отбрасывания дробной части. Таким образом мы обрабатываем числа напрямую, сохраняя ленивость у последовательности. Далее выводим каждое значение последовательности с помощью Seq.iter, данная функция применяет printf к каждому значению последовательности. Программа завершает работу с кодом возврата 0.

# Задание 2

## Задача 1

### Текст задачи

Последовательность содержит строки. Сколько из них имеет чётную длину?



### Алгоритм решения

Программа начинает свое выполнение с функции main. Запрашивается у пользователя количество элементов последовательности, которое сохраняется в значение count. Затем введенное значение преобразуется в целое число. Выполняется проверка: если count меньше или равно 0, выводится сообщение «Ошибка: количество должно быть положительным!» и программа завершается с ошибкой через exit 1. Если count положительное число, вызывается функция readStringsSeq для формирования последовательности строк. Данная функция создает последовательность с помощью оператора seq, и запускается цикл for от 1 до count, где count это корректные данные. В каждой итерации цикла выводится сообщение «Введите элемент index: » с текущим номером элемента. Считывается строка, введенная пользователем, и полученное значение добавляется в формируемую последовательность с использованием yield. После завершения цикла сформированная последовательность строк возвращается в функцию main и сохраняется в значение stringsSeq. Затем с помощью функции Seq.fold выполняется одновременно два действия: формирование строки для вывода всех элементов и подсчет количества строк с четной длиной. Аккумулятор представляет собой кортеж из строки и целого числа. Для каждой строки s проверяется, делится ли ее длина на 2 без остатка (s.Length % 2 = 0), и если условие выполняется, счетчик увеличивается на 1, а строка добавляется к результирующей строке. Результат сохраняется в кортеж (resultString, evenCount). В конце программы на экран выводятся все строки последовательности и количество строк с четной длиной. Программа завершает работу с кодом возврата 0.

# Задание 3

## Задача 1

### Текст задачи

Выяснить, есть ли в каталоге файл с введенным названием.



### Алгоритм решения

Программа начинает свое выполнение с функции main. Запрашивается у пользователя путь к каталогу, который сохраняется в значение directoryPath. Затем с помощью метода Directory.Exists выполняется проверка существования указанного каталога. Если каталог не существует, выводится сообщение «Ошибка: Каталог 'путь' не существует!» и программа ожидает нажатия клавиши для выхода. Если каталог существует, запрашивается название файла для поиска, которое сохраняется в значение searchFileName.



Для получения списка файлов используется функция getFileNames, которая создает последовательность с помощью оператора seq. В этой функции метод Directory.EnumerateFiles возвращает перечисляемую коллекцию полных путей всех файлов в указанном каталоге. Для каждого полного пути с помощью Path.GetFileName извлекается только имя файла, которое добавляется в последовательность с использованием yield. Сформированная последовательность имен файлов возвращается и сохраняется в значение filesSeq.



Для вывода всех файлов запускается цикл for, который перебирает элементы последовательности filesSeq и выводит каждое имя файла с префиксом " - ".



Для проверки существования искомого файла вызывается функция checkFileExists, которая принимает путь к каталогу и имя файла. Внутри этой функции с помощью getFileNames получается последовательность имен файлов, и к ней применяется функция Seq.exists, проверяющая наличие элемента, равного искомому имени. Результат проверки (true или false) возвращается и сохраняется в значение fileExists.



В зависимости от результата проверки выводится соответствующее сообщение: если файл существует — «Файл 'имя' существует!», если не найден — «Файл 'имя' не найден.». В конце программа ожидает нажатия любой клавиши для выхода и завершается с кодом возврата 0.
